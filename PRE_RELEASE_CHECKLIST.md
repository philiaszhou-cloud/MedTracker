## 🚨 常见问题阻挡

### ❌ 不能通过的情况

```
✗ 编译失败 (Exit code != 0)
  → 不能打包，必须先修复 TypeScript 错误

✗ 功能缺陷 (某个测试失败)
  → 不能发布，必须先修复功能 bug

✗ UI 卡顿 (> 30s 无响应)
  → 不能发布，用户体验严重受影响

✗ 安全问题 (打印了敏感信息)
  → 不能发布，必须先修复安全问题

✗ TypeScript 类型错误 (有 any 转换)
  → 理论上可以，但建议修复提高代码质量
```

### ⚠️ 需要注意的情况

```
⚠️  日志太多 (影响性能)
  → 建议注释掉部分 debug 日志

⚠️  APK 大小过大 (> 50MB)
  → 建议检查是否有冗余依赖

⚠️  某个场景超时很长 (接近 25s)
  → 建议优化，但不能超过 25s

⚠️  日志中有警告 (但功能正常)
  → 监控，但不一定需要修复
```
